### 一、 核心开发提示词（System Prompt）

**Role:** 你是一位全栈架构师，擅长 UI/UX 设计与 React 技术栈。

**Project:** “Drink&Chill” - 一个专为年轻职场人打造的酒类探索与心情管理平台。

**Tech Stack:** React (Next.js), Tailwind CSS (样式), Framer Motion (交互动画), SQLite (本地存储), PWA (支持离线)。

**Requirements:**

1、**架构模式:** Client-Server 架构。前端负责交互，后端使用 API 路由处理数据缓存与外部服务请求。

2、**核心功能:**

- **首页:** 参考我提供的首页设计图，地图上可以先重点标注中、日、韩、英、法、德等国家。首页中间上方提供“心情输入”入口，系统可根据心情推荐合适的酒（支持预设标签 + 自然语言 AI 转换）。

- **页面左侧竖排导航栏**:
  - 1、推荐 (根据心情、目标口味推荐；推荐最新款式、应季产品，推荐的款式旁要附上调酒所需的材料和制作过程)。
  - 2、分类 (原料/味道/度数/国家（点击“国家”时，会打开世界地图页面，在地图上点击不同国家可显示“上方是该国家的美景图片，下面是该国代表性的基酒和相关产品，酒的起源故事等等“。)
  - 3、附近 (提供附近酒吧、便利店位置)。
  - 4、灵感：（在这里可记录灵机一动的想法，是私密空间）。

- **互动:** 使用浮窗/Modal 进行教程展示。

- **地图:** 使用 Mapbox 实现“起源国家”的交互地图，点击国家弹出美景图、起源故事、产品库。

- **存储:** 使用 SQLite 实现“灵感”数据的本地持久化（图片+心情标签+配方备份）。

- **合规:** 首页强制弹出“法定饮酒年龄验证”组件。

- **交付标准:** 代码需模块化，项目目录清晰有逻辑；UI风格为大面积白色铺底，辅助颜色为黑色、浅蓝色。
